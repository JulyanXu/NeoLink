# Hermes Agent 维护说明

Hermes 的职责是维护内容，不负责重构 UI。每日更新优先只改 `data/market-news.json`，必要时同步更新 `data/source-watchlist.json`、`data/taxonomy.json` 和 `assets/logos/`。如需改 UI，先阅读 `docs/frontend-adjustment.md`。

完整交接入口见 `docs/hermes-handoff.md`。

## 每日流程

1. 采集来源：政府监管、地方平台、电力交易中心、政务信用公示、招投标、协会报告、国际组织、企业公开信息、财经媒体、行业媒体、商业数据库、授权调研和用户贡献。
2. 去重归类：同一事件只保留一条主记录，后续进展改摘要、状态或新增进展标签。
3. 核验来源：媒体报道、社交媒体、用户投稿和招聘/专利等间接信息先进入线索池；没有公告、监管文件、企业原文、权威数据库、授权材料或第二可信来源的内容，`status` 必须标为 `待核验` 或 `跟踪中`。
4. 写入数据：更新 `generatedAt`，新增情报插入 `news` 数组顶部。
5. 运行校验：`npm run build`。
6. 发布部署：把静态文件部署到托管平台。

页面会按 `news.date` 倒序展示每日简报，所以 JSON 维护顺序不必强制手动排序。

## 内容要求

每条 `news` 至少包含：

- `id`：稳定唯一 ID，建议 `yyyymmdd-category-slug`
- `date`：事件日期，格式 `YYYY-MM-DD`
- `category`：见 `data/taxonomy.json`
- `priority`：`high`、`medium`、`low`
- `impact`：`bullish`、`bearish`、`neutral`、`mixed`、`watch`
- `region`：全国、地区、海外、资本市场、产业链等
- `title`：一句话事实标题，不写夸张判断
- `summary`：说明事件、口径、影响和下一步观察点
- `source`：来源名称
- `sourceUrl`：原文链接，没有就留空并标记待核验
- `status`：`已核验`、`待核验`、`跟踪中`
- `tags`：数组，便于检索

## 排行榜与 logo

- 排行榜维护在 `data/market-news.json` 的 `companyTrackers`。
- `title` 和 `basis` 必须写明年份、榜单口径、来源机构和授权限制。
- 企业 logo 优先放在 `assets/logos/`，并在 `company.logoUrl` 中使用相对路径，例如 `assets/logos/svolt.png`。
- 不要热链不稳定的搜索引擎缩略图；可以用 Bing 图片搜索定位来源，但落地时优先下载企业官网、公告页、展会页或可公开访问的品牌资源。
- 新增或替换 logo 后运行 `npm run build`，校验脚本会检查本地 logo 文件是否存在。

## 禁止事项

- 不把未证实传言写成事实。
- 不把单一媒体报道直接当作最终事实，除非该媒体原文已经引用了可核验文件或权威数据源。
- 不把社交媒体、论坛、群聊截图和未经证实的自媒体内容作为单一来源。
- 不破解、不绕过登录、不抓取非公开接口、不暴力爬取、不保存付费数据库账号、Cookie、API key 或原始导出数据。
- 不发布涉密数据、个人隐私数据、非法购买数据或未授权商业数据。
- 不删除仍有追踪价值的旧内容，除非确认为重复或错误。
- 不在仓库写入平台账号、Cookie、API key 或付费数据库凭据。
- 不直接修改 UI 文件，除非用户明确要求。

## 来源分层

- 一手来源：国家部委、地方政府平台、电力交易中心、政务信用公示、交易所披露、企业公告、法院/招标/海关文件、正式授权数据。
- 二手来源：行业协会、国际组织、第三方研究机构、财经媒体、证券媒体、行业媒体、海外能源媒体和研究机构报告。
- 线索来源：企业公众号、专家专栏、招聘信息、专利、投资者问答、社交媒体、论坛、用户投稿和爆料。

媒体和线索来源可以提高覆盖面，但必须在 `summary` 中写清楚口径和不确定性。

## 六类来源执行表

1. 政府及监管机构：国家能源局、发改委、工信部、国家统计局、地方发改委/能源局/自然资源厅、电力交易中心、国家企业信用信息公示系统、中国招标投标公共服务平台。优先用于政策、装机、发电量、项目备案、处罚、招标和消纳数据。
2. 行业协会与第三方研究机构：CPIA、CWEA、中国能源研究会、充电联盟、IEA、IRENA、GGII、CINNO、中电联、赛迪、艾瑞。优先用于白皮书、产量、装机、进出口、技术路线图、企业排名和季度/年度报告。
3. 企业公开信息：巨潮资讯、上交所、深交所、官网、公众号、ESG报告、投资者关系记录、工商年报、招投标、专利、新闻稿和招聘信息。只抓公开页面，不登录、不破解、不抓非公开接口。
4. 专业数据库与媒体平台：北极星电力网、中国能源报、能源世界、财新、第一财经、天眼查、企查查、BloombergNEF、S&P Global、ICIS。商业数据必须核对版权和API授权。
5. 合规一手信息：协会/企业/数据公司授权、官方开放API、行业访谈、问卷调研、用户投稿、展会和发布会公开资料。投稿和调研需匿名、自愿、不侵犯隐私。
6. 法律红线：禁止黑客、破解、暴力爬虫、非法购买涉密/隐私数据；必须注明来源、保护个人信息、遵守《著作权法》《网络安全法》《数据安全法》。

## 第三方咨询授权规则

- GGII、CINNO Research、中电联、赛迪顾问、艾瑞咨询等机构的公开新闻页、目录页、报告摘要页可以作为线索入口。
- 年度报告、季度报告、企业排名明细、细分市场数据、价格库、数据库导出和付费 PDF 只能在授权范围内引用。
- 写入 `data/market-news.json` 时，`summary` 必须标注数据机构、统计周期、统计口径和授权状态；无法确认授权时只写公开摘要，不写完整表格或明细排名。
- 不保存账号、Cookie、付费报告原文、截图、导出表或未经授权的数据字段。
- 商业发布前必须确认合同允许的引用格式、字段范围、展示方式和使用期限。

## 发布前检查

```bash
npm run build
```

正式发布不允许存在 `示例/待替换` 内容。上线前用严格模式拦截：

```bash
STRICT_CONTENT=1 npm run build
```
